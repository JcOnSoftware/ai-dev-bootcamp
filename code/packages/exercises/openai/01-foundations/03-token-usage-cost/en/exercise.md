# Exercise 03 — Token usage and cost calculation

## Concept

Every API call consumes **tokens** — the basic unit of text processing for LLMs. A token is roughly 4 characters or ¾ of a word in English.

The OpenAI response includes a `usage` field with:
- `prompt_tokens`: tokens consumed by your message + system context
- `completion_tokens`: tokens generated by the model in the response
- `total_tokens`: the sum of both

The actual cost is calculated by multiplying those counts by the model's rate. For `gpt-4.1-nano`:
- Input: **$0.10 per 1,000,000 tokens**
- Output: **$0.40 per 1,000,000 tokens**

Formula: `(prompt_tokens * 0.10 + completion_tokens * 0.40) / 1_000_000`

Understanding token usage is essential for estimating budgets, choosing models, and optimizing prompts.

## Your task

1. Open `starter.ts`.
2. Create an OpenAI client and make a call with `gpt-4.1-nano` and `max_completion_tokens: 128`.
3. Extract token usage from `response.usage`.
4. Calculate the cost in USD using this formula:
   ```typescript
   const cost = (usage.prompt_tokens * 0.10 + usage.completion_tokens * 0.40) / 1_000_000;
   ```
5. Return `{ response, cost }`.

## Extra concept (optional)

You can count tokens **before** making a call using the `tiktoken` library (OpenAI's official tokenizer). This is useful for:
- Estimating costs before calling the API
- Verifying your prompt doesn't exceed the model's context limit
- Optimizing prompt length

```typescript
import { encoding_for_model } from "tiktoken";
const enc = encoding_for_model("gpt-4o");
const tokens = enc.encode("Your text here");
console.log(tokens.length); // token count
enc.free();
```

In practice, GPT-4 models use `cl100k_base` encoding, while GPT-4o and GPT-4.1 use `o200k_base`.
